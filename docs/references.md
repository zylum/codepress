# References & Inspiration

## Predecessor

| Resource | Description |
|---|---|
| [Linotype](https://github.com/zylum/linotype) | The predecessor to CodePress. A CLI-only tool for product development. The naming conventions (Galley, Slug) originate here. |

## Related methodologies

| Resource | Description |
|---|---|
| [Compound Engineering](https://every.to/guides/compound-engineering) | An AI delivery methodology by Dan Shipper at Every. CodePress complements Compound Engineering with product shaping and knowledge capture. |
| [BMAD Method](https://github.com/bmad-code-org/bmad-method) | A build-measure-learn approach for software. Focuses on validated learning through iterative delivery. |
| [Shape Up](https://basecamp.com/shapeup) | Basecamp's product shaping methodology. Shapes before building, sets appetite instead of estimates. CodePress's Galley stage is influenced by Shape Up. |
| [Value Over Everything](https://www.linkedin.com/newsletters/value-over-everything-7015605132717359104/) | A newsletter on product and value by Rich Mironov. |
| [Cynefin Framework](https://en.wikipedia.org/wiki/Cynefin_framework) | A sense-making framework for decision-making. CodePress autonomy levels (continuous/confirm/stop) map to Cynefin domains. |

## Tools

| Resource | Description |
|---|---|
| [Google Slides API](https://developers.google.com/slides) | Used by the SlideMaker MCP for deck creation |
| [Model Context Protocol](https://modelcontextprotocol.io) | The protocol that powers Codepress's AI integration |
| [Iconify](https://iconify.design) | Free icon API used by SlideMaker for slide icons |
