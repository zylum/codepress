## What Domain Memory is for

Domain Memory is the shared operating context for humans and AI.
It answers: *what does this system do and how does it work?*

It is not exhaustive documentation.
It is the minimum context needed to work without re-discovering everything.

---

## Keeping it current

- Add to `concepts.md` when a new concept is introduced
- Add to `glossary.md` when a term needs defining
- Update `current-state.md` when capabilities change significantly
- Add to `decisions/` when a significant architectural or product decision is made

Domain Memory grows from delivery, not from planning.
Prefer a 5-line stub over nothing. Update incrementally.
